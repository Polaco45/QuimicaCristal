POS: Payment Reference Details

Description

POS: Payment Reference Details module enhances the Odoo Point of Sale (POS) system by enabling cashiers and users to enter a payment reference when processing payments through card, UPI, or other digital payment providers. Odoo's default POS system lacks a built-in functionality to capture payment reference details, which is crucial for validating transactions.

Key Features:
Payment Reference Entry: Requires cashiers to input a payment reference number when processing payments via digital payment methods.
Easily Toggle Functionality: Enable or disable the payment reference requirement in POS settings to suit your business needs.
Payment Reference in Payment Lines: Easily keep track of payment details by storing references directly in your POS transactions.
Audit & Record Keeping: Stores payment references for future verification and reconciliation.

Benefits:
Ensures payment validation before completing transactions.
Reduces discrepancies and fraud risks in digital payments.
Improves financial record accuracy for audits and reconciliation.

This module is a must-have for businesses using digital payment methods in Odoo POS, ensuring secure and verified transactions.

Installation & Configuration
Configuration:
After installing the POS: Payment Reference Details module, a configuration page will be available for each POS store. You can configure the Payment Reference Details settings from Configurations > Settings in the POS store configuration.
Enabling the Allow Payment Reference for Payment Methods setting for a POS store lets you manage payment reference details for selected payment methods in POS transactions. This helps ensure accurate payment validation, reduce errors, and improve transaction tracking for better financial reconciliation.
When a customer makes a payment using a method that requires a payment reference, a Payment Reference button will be available for that transaction. Clicking this button opens a popup where the cashier/user can enter the payment reference details.
After entering the payment reference details, click the Confirm button. Upon validating the payment, the entered reference details will be stored in the POS Payment record in the backend.
